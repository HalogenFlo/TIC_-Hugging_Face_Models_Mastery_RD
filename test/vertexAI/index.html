Chiến Lược Tiết Kiệm Chi Phí Cực Hạn (Cost-Saving Strategy)

Triển khai mô hình AI lên Cloud thường rất tốn kém (hàng chục đến hàng trăm USD/tháng do chi phí GPU). Để giải quyết bài toán ngân sách hạn hẹp của học viên và doanh nghiệp nhỏ, chúng ta sẽ áp dụng 4 nguyên lý tối ưu chi phí cực hạn sau:

⚠️ Lãng Phí Nguy Hiểm
  • Thuê GPU (T4, L4, A100) để test mô hình nhỏ, tiêu tốn $0.35 - $3.50 mỗi giờ vô ích.
  • Đóng gói Docker Image khổng lồ chứa CUDA (5GB - 12GB) làm tăng phí lưu trữ trên Artifact Registry.
  • Để quên Endpoint chạy 24/7 sau khi hoàn thành bài tập, phát sinh hóa đơn hàng trăm USD vào cuối tháng.
💡 Giải Pháp Tiết Kiệm 95% Chi Phí
  • Chạy CPU-only: Mô hình Qwen2.5-0.5B cực kỳ nhỏ (~1GB), chạy mượt mà trên máy ảo CPU rẻ tiền (như n1-standard-2).
  • Docker Slim Image: Dùng image base Python siêu nhẹ và chỉ cài đặt PyTorch phiên bản CPU để giảm dung lượng Docker xuống <1.2GB.
  • Tự động Hủy Tài Nguyên (Undeploy): Quy tắc vàng: Deploy -> Kiểm thử nhanh -> Undeploy & Xóa Endpoint ngay lập tức (chỉ tốn vài cent phí chạy thử).
Chìa Khóa Kỹ Thuật: GCP Storage FUSE

Vertex AI tự động liên kết (mount) thùng chứa Cloud Storage gs://your-bucket thành thư mục nội bộ /gcs/your-bucket bên trong Container Docker khi deploy. Nhờ vậy, container chỉ cần đọc weights trực tiếp từ ổ đĩa nội bộ mà không cần viết code tải file phức tạp, giúp giảm thiểu tối đa băng thông mạng (Data Egress) và giảm dung lượng RAM cần thiết.

1. Chuẩn Bị & Xác Thực Google Cloud (GCP)

Để lập trình và giao tiếp với Google Cloud Platform, hãy thực hiện cài đặt các thư viện và xác thực tài khoản:

Bước 1.1: Cài đặt Python SDK cho Vertex AI

Cài đặt các gói thư viện Python chính thức từ Google Cloud để quản lý model registry và endpoints:

bash 
pip install google-cloud-aiplatform python-dotenv fastapi uvicorn
Bước 1.2: Xác thực tài khoản Google Cloud dưới terminal

Chạy các lệnh xác thực sau để cấp quyền cho code Python nội bộ gọi API lên GCP:

bash 
# 1. Đăng nhập tài khoản Google Cloud của bạn
gcloud auth login

# 2. Cấu hình credentials mặc định cho các ứng dụng Python
gcloud auth application-default login

# 3. Đặt dự án mặc định của bạn
gcloud config set project [PROJECT_ID_CỦA_BẠN]
Bước 1.3: Cấu hình biến môi trường (File .env)

Tạo file cấu hình tập trung để lưu trữ các thông tin hạ tầng GCP:

env 
# --- CẤU HÌNH HẠ TẦNG GCP ---
GCP_PROJECT_ID=ticmastery-project
GCP_REGION=us-central1
GCP_BUCKET_URI=gs://tic-mastery-bucket
HUGFACE_TOKEN=hf_your_access_token_here

2. Cách 1: Triển Khai Nhanh qua Deep Learning Container (DLC)

Phương pháp triển khai nhanh sử dụng Container tối ưu hóa sẵn của Google (Hugging Face Deep Learning Container) để deploy trực tiếp một mô hình nền tảng từ Hugging Face Hub (ví dụ: Qwen/Qwen2.5-0.5B-Instruct) lên một máy ảo CPU giá rẻ.

Bí Quyết Tiết Kiệm

Trong tham số model.deploy(), hãy chỉ định machine_type="n1-standard-2" (hoặc n1-standard-4) và hoàn toàn không khai báo GPU (như accelerator_type). Máy ảo CPU của GCP có chi phí cực kỳ rẻ (chỉ khoảng ~$0.09 - $0.19 mỗi giờ), giúp bạn thoải mái thử nghiệm mà không lo cạn kiệt tài khoản.

python 
import os
from google.cloud import aiplatform

# Khởi tạo Vertex AI SDK với project và region của bạn
aiplatform.init(
    project=os.environ.get("GCP_PROJECT_ID", "ticmastery-project"),
    location=os.environ.get("GCP_REGION", "us-central1")
)

# 1. Đăng ký mô hình sử dụng HF Deep Learning Container của Google (CPU serving)
model = aiplatform.Model.upload(
    display_name="hf-qwen-cpu-model",
    # Sử dụng image serving CPU tiêu chuẩn để giảm thiểu chi phí phần cứng
    serving_container_image_uri="us-docker.pkg.dev/deeplearning-platform-release/gcs-fuse/huggingface-text-generation-inference.1-3.py310",
    serving_container_environment_variables={
        "HF_MODEL_ID": "Qwen/Qwen2.5-0.5B-Instruct",
        "HF_TASK": "text-generation",
        "MAX_INPUT_LENGTH": "1024",
        "MAX_TOTAL_TOKENS": "2048"
    }
)

print("👉 Đang tạo endpoint...")
endpoint = aiplatform.Endpoint.create(display_name="qwen-cpu-endpoint")

# 2. Deploy lên máy ảo CPU giá rẻ n1-standard-2 (Không sử dụng GPU!)
print("🚀 Đang deploy mô hình lên Endpoint CPU (Thời gian deploy khoảng 5-10 phút)...")
model.deploy(
    endpoint=endpoint,
    machine_type="n1-standard-2", # Cấu hình 2 vCPUs, 7.5GB RAM - Tiết kiệm tối đa!
    min_replica_count=1,
    max_replica_count=1
)
print(f"✅ Đã deploy thành công! Endpoint ID: {endpoint.resource_name}")

3. Cách 2: Triển Khai Tùy Biến (Custom Serving Container)

Áp dụng khi bạn đã tinh chỉnh mô hình (QLoRA) và muốn đóng gói một API Server FastAPI tùy biến logic riêng. Container được tối ưu hóa cực kỳ mỏng nhẹ chạy độc quyền trên CPU để tiết kiệm dung lượng Artifact Registry và RAM hệ thống.

Bước 3.1: Viết API Server FastAPI tối ưu CPU (app/main.py)

Code FastAPI được thiết kế tự động đọc weights được mount từ GCS thông qua FUSE và tự động cấu hình chạy trên CPU:

python 
import os
import torch
from fastapi import FastAPI, Request
from transformers import AutoTokenizer, AutoModelForCausalLM

app = FastAPI()

# 1. TỰ ĐỘNG ĐỌC MÔ HÌNH QUA GCS FUSE:
# Vertex AI sẽ tự động lưu đường dẫn bucket mount vào biến môi trường AIP_STORAGE_URI
GCS_URI = os.environ.get("AIP_STORAGE_URI", "")
MODEL_PATH = GCS_URI.replace("gs://", "/gcs/") if GCS_URI.startswith("gs://") else "/workspace/model"

print(f"👉 [Vertex AI Serving] Nạp model weights từ GCS FUSE: {MODEL_PATH}")

# 2. Nạp Tokenizer và Model
tokenizer = AutoTokenizer.from_pretrained(MODEL_PATH)
# Nạp mô hình ở chế độ float32/float16 tối ưu cho suy luận CPU
model = AutoModelForCausalLM.from_pretrained(
    MODEL_PATH, 
    torch_dtype=torch.float32, 
    low_cpu_mem_usage=True
)

# Khóa cứng chạy trên CPU để tránh tìm kiếm thư viện CUDA gây crash
device = "cpu"
model.to(device)
print(f"✅ Hệ thống phục vụ chạy thành công trên: {device}")

@app.get("/health")
def health_check():
    # Điểm kiểm tra sức khỏe bắt buộc của Vertex AI (Health Check)
    return {"status": "healthy"}

@app.post("/predict")
async def predict(request: Request):
    # API nhận định dạng JSON chuẩn của Vertex AI: {"instances": ["câu hỏi 1", "câu hỏi 2"]}
    body = await request.json()
    instances = body.get("instances", [])
    results = []
    
    for text in instances:
        inputs = tokenizer(text, return_tensors="pt").to(device)
        with torch.no_grad():
            outputs = model.generate(**inputs, max_new_tokens=30)
            response_text = tokenizer.decode(outputs[0], skip_special_tokens=True)
        results.append(response_text)
        
    # Vertex AI yêu cầu phản hồi dạng {"predictions": [kết quả 1, kết quả 2]}
    return {"predictions": results}
Bước 3.2: Viết Dockerfile tối ưu dung lượng (Mỏng Nhẹ)

Sử dụng base image python:3.10-slim và cài đặt PyTorch phiên bản CPU để dung lượng Docker Image đạt mức siêu nhỏ (~1.1GB), giúp giảm chi phí lưu trữ trên Artifact Registry:

dockerfile 
FROM python:3.10-slim

WORKDIR /workspace
COPY ./app /workspace/app

# Cài đặt PyTorch CPU-only từ source wheel chính thức để giữ dung lượng image siêu nhỏ gọn
RUN pip install --no-cache-dir --index-url https://download.pytorch.org/whl/cpu \
    torch torchvision torchaudio \
    && pip install --no-cache-dir fastapi uvicorn transformers huggingface_hub

# Cổng 8080 là cổng mặc định phục vụ của Vertex AI
EXPOSE 8080

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8080"]
Bước 3.3: Đóng gói và đẩy Container Image lên Google Artifact Registry (GAR)

Biên dịch Docker image cục bộ và đẩy lên kho lưu trữ bảo mật của Google Cloud:

bash 
# 1. Tạo Docker Repository trên GCP (nếu chưa có)
gcloud artifacts repositories create custom-models-repo \
    --repository-format=docker --location=us-central1 --description="Repository chứa mô hình CPU"

# 2. Xác thực Docker daemon với Google Artifact Registry
gcloud auth configure-docker us-central1-docker.pkg.dev

# 3. Build Docker image siêu nhẹ
docker build -t custom-cpu-predictor:v1 .

# 4. Gán nhãn trỏ về Artifact Registry của dự án
docker tag custom-cpu-predictor:v1 us-central1-docker.pkg.dev/[PROJECT_ID_CỦA_BẠN]/custom-models-repo/my-cpu-predictor:v1

# 5. Đẩy image lên cloud
docker push us-central1-docker.pkg.dev/[PROJECT_ID_CỦA_BẠN]/custom-models-repo/my-cpu-predictor:v1
Bước 3.4: Đăng ký & Triển khai lên Endpoint CPU thật

Đẩy weights mô hình tinh chỉnh lên GCS Bucket, đăng ký custom container và deploy lên Endpoint máy ảo CPU rẻ nhất:

python 
# 1. Tải weights mô hình lên GCS Bucket (Chạy ở Terminal):
# gcloud storage cp -r ./notebook/Qwen2.5-0.5B-Instruct-lora-finetuned gs://[BUCKET_NAME]/models

# 2. Tập lệnh Python đăng ký Custom Model & Deploy lên Vertex CPU Endpoint:
import os
from google.cloud import aiplatform

aiplatform.init(
    project=os.environ.get("GCP_PROJECT_ID", "ticmastery-project"),
    location=os.environ.get("GCP_REGION", "us-central1")
)

# Đăng ký mô hình tùy biến liên kết với đường dẫn weights trên GCS
model = aiplatform.Model.upload(
    display_name="qwen-custom-cpu-model",
    # Đường dẫn thư mục weights trên GCS (sẽ được tự động mount vào /gcs/)
    artifact_uri="gs://[BUCKET_NAME]/models",
    serving_container_image_uri="us-central1-docker.pkg.dev/[PROJECT_ID_CỦA_BẠN]/custom-models-repo/my-cpu-predictor:v1",
    serving_container_ports=[8080]
)

print("👉 Đang tạo endpoint...")
endpoint = aiplatform.Endpoint.create(display_name="custom-cpu-endpoint")

# Deploy lên máy ảo CPU giá rẻ n1-standard-2 để tiết kiệm chi phí cực hạn
print("🚀 Đang tiến hành deploy custom container (khoảng 5-10 phút)...")
model.deploy(
    endpoint=endpoint,
    machine_type="n1-standard-2", # Chỉ tốn khoảng $0.09/giờ!
    min_replica_count=1,
    max_replica_count=1
)
print(f"✅ Đã deploy thành công! Endpoint ID: {endpoint.resource_name}")

4. Kiểm Thử Endpoint & Hủy Tài Nguyên (Dọn Dẹp)

Sau khi deploy thành công, hãy thực hiện kiểm thử nhanh dữ liệu dự đoán và thực hiện **quy trình dọn dẹp bắt buộc** ngay lập tức để tránh phát sinh chi phí duy trì máy ảo ngầm ngoài ý muốn.

Bước 4.1: Kiểm thử gửi yêu cầu suy luận (Inference)

Sử dụng Python SDK để gọi API suy luận thông qua Endpoint Gateway bảo mật của Google:

python 
from google.cloud import aiplatform

# Cấu hình ID endpoint vừa tạo của bạn
ENDPOINT_ID = "projects/[PROJECT_ID]/locations/us-central1/endpoints/[ENDPOINT_NUMERIC_ID]"

endpoint = aiplatform.Endpoint(endpoint_name=ENDPOINT_ID)

# Gửi yêu cầu suy luận theo định dạng instances chuẩn
response = endpoint.predict(
    instances=["What is the capital of Vietnam?"]
)

print("👉 Kết quả dự đoán từ Vertex AI CPU Endpoint:")
print(response.predictions)
Bước 4.2: Dọn dẹp triệt để tránh bị tính phí ngầm (QUAN TRỌNG NHẤT)

Mô hình đăng ký trên Registry thì không tốn tiền duy trì, nhưng máy ảo liên kết với Endpoint thì tính tiền theo giờ. Hãy chạy đoạn script Python dọn dẹp tự động này ngay sau khi test xong:

python 
from google.cloud import aiplatform

# Khởi tạo kết nối tới Endpoint của bạn
ENDPOINT_ID = "projects/[PROJECT_ID]/locations/us-central1/endpoints/[ENDPOINT_NUMERIC_ID]"
endpoint = aiplatform.Endpoint(endpoint_name=ENDPOINT_ID)

print("⚠️ Đang tiến hành hủy deploy tất cả các mô hình khỏi máy ảo...")
# 1. Hủy deploy mô hình để tắt VM Instance bên dưới Endpoint
endpoint.undeploy_all()

print("🧹 Đang xóa Endpoint...")
# 2. Xóa Endpoint để giải phóng tài nguyên mạng hoàn toàn
endpoint.delete()

print("✅ Đã dọn dẹp sạch sẽ! Hệ thống của bạn hiện tại an toàn 100% không lo phát sinh hóa đơn phát sinh.")
