2. Cấu hình Hệ thống (config.py)

Tệp config.py đóng vai trò là trung tâm quản lý cấu hình môi trường, khởi tạo driver kết nối cơ sở dữ liệu và chuyển đổi giao tiếp mô hình LLM (Adapters).

2.1. Quản lý Kết nối Cơ sở dữ liệu (Database Drivers)

  • Neo4j Database: Quản lý qua hàm get_neo4j_driver(), cung cấp instance kết nối đồ thị duy nhất và tự động đóng thông qua close_neo4j_driver().
  • Redis Client: Hàm get_redis_client() khởi tạo kết nối đến Redis, dùng để lưu giữ bộ nhớ phiên làm việc ngắn hạn (short-term session memory).
  • PostgreSQL / Relational Connection: Hàm get_db_connection() tự động lựa chọn adapter kết nối: ưu tiên SQLAlchemy Engine nếu có cấu hình URI, hoặc tự động fallback về thư viện psycopg2 (cho Postgres) hoặc pymysql (cho MySQL) để lưu trữ bộ nhớ dài hạn của người dùng.

2.2. Giao thức tương thích LLM (Model Adapters)

Để hệ thống Multi-Agent có thể hoạt động độc lập với nhà cung cấp mô hình (Model-Agnostic), config.py cung cấp hai adapter chuyển đổi trung gian:

OpenAIModelAdapter

Sử dụng SDK OpenAI để tương thích các dòng GPT. Hỗ trợ beta.chat.completions.parse để bắt buộc ép cấu trúc JSON Schema (Structured Output) thông qua Pydantic.

GeminiModelAdapter

Tích hợp SDK Google GenAI mới nhất, chuyển đổi định dạng danh sách hội thoại thô của LangGraph sang cấu trúc contents và system_instruction riêng biệt của Google.

3. Bộ nhớ Toàn cục & State Management (state.py)

Tệp state.py định nghĩa lớp AgentState dùng chung cho toàn bộ đồ thị LangGraph, đóng vai trò là "bảng tin trung tâm" để các Agent ghi nhận thông tin và chuyển tiếp dữ liệu cho nhau.

3.1. Các Reducers gộp dữ liệu song song (Parallel Reducers)

Khi các Sub-Agent chạy song song (Fan-out/Fan-in), LangGraph mặc định sẽ ghi đè biến trạng thái. Để giải quyết, state.py định nghĩa hai hàm gộp dữ liệu đặc biệt:

  • merge_dict(left, right): Gộp lũy kế các cặp Key-Value từ các Agent khác nhau vào một dictionary trạng thái chung (dùng cho domain_outputs, evaluation) thay vì ghi đè lẫn nhau.
  • merge_list(left, right): Cộng dồn danh sách các phát hiện xung đột và trích dẫn nguồn từ các nhánh chạy song song khác nhau (dùng cho detected_domains, conflicts, citations).
