Quyết định Kiến trúc & Quy chuẩn Code

Các quy tắc ràng buộc kỹ thuật bắt buộc để AI và lập trình viên tuân thủ trước khi triển khai dự án.

1. Tổng quan Quyết định Kiến trúc

Hạng mục Quyết định thống nhất Lý do & Ý nghĩa ràng buộc
Framework điều phối LangGraph (StateGraph) Quản lý đồ thị luồng suy luận của tác nhân, hỗ trợ state management tập trung, conditional edges, retry, checkpoint.
CSDL Vector & Đồ thị Neo4j Vector Index Duyệt kết hợp tìm kiếm ngữ nghĩa (vector) và duyệt mối quan hệ đồ thị (GraphRAG) trực tiếp trên Neo4j để tối ưu hiệu năng.
Cơ chế Song song hóa Động theo yêu cầu Coordinator phân tích câu hỏi động để xác định các domain luật liên quan (Thuế, Đất đai...). Nếu có nhiều domain, tự động spawn song song các Law Agent tương ứng.
Conflict Agent Tách riêng, luôn chạy Chạy cho cả trường hợp 1 domain (giải quyết mâu thuẫn nội tại Luật > Nghị định > Thông tư) và trường hợp nhiều domain (kiểm tra xung đột chéo).
Mở rộng domain Mỗi Agent = 1 file riêng Mỗi agent con viết code độc lập, system instruction riêng biệt trong file riêng để dễ mở rộng và cô lập lỗi, tuyệt đối không dùng class chung tham số hóa.
Đồng bộ hóa Schema CSDL Bắt buộc theo CSDL thực tế Mọi truy vấn Cypher và so khớp ngữ nghĩa phải tuân thủ nghiêm ngặt schema thực tế của đồ thị Neo4j (Ví dụ: dùng norm_number thay vì code, accumulated_text thay vì text, và chỉ MATCH các mối quan hệ thực tế như CITES|IMPLEMENTS|REFERS_TO).

3. Nguyên tắc Code Khoa học & Dễ quản lý

Ràng buộc Quy chuẩn lập trình bắt buộc đối với AI / Dev
  • Cô lập chức năng (Modularity):
    • Tools (Tầng 1): Chỉ tương tác thô với CSDL/Môi trường, không xử lý logic nghiệp vụ hay suy luận AI.
    • Skills (Tầng 1): Đóng gói quy trình gọi các tool thô, chịu trách nhiệm xử lý dữ liệu trung gian.
    • Agents (Tầng 2): Nhận dữ liệu sạch đã được chuẩn bị từ các Skill để lập luận, suy nghĩ và trả kết quả dưới dạng JSON Schema.
  • Functional Programming (FP):
    • Tuyệt đối tránh sử dụng let/var ở các biến trạng thái nếu có thể dùng const (hoặc immutability trong Python).
    • Hạn chế side-effects, các hàm xử lý dữ liệu phải là các pure functions để dễ dàng viết Unit Test.
    • Tránh lạm dụng OOP (Class phức tạp), ưu tiên viết các helper function nhỏ gọn.
  • Hợp đồng 4 phần Agent (BaseAgent): Mọi Agent chuyên môn được tạo ra bắt buộc phải tuân thủ interface BaseAgent với 4 phần độc lập:
    1. Goal & Context: Mục tiêu cụ thể và instruction đầu vào.
    2. Task Boundary: Phạm vi được phép xử lý của Agent.
    3. Skills & Tools: Danh sách công cụ được cấp quyền sử dụng.
    4. Output Schema: Cấu trúc JSON Schema bắt buộc của đầu ra.
  • Cá nhân hóa & Bộ nhớ dài hạn (Long-term Personal Memory):
    • Phân tách rạch ròi theo Thread-based (chủ đề hội thoại), chia sẻ chung 1 User Profile lưu ở PostgreSQL (sử dụng cột JSONB để lưu trữ linh hoạt hai trường facts và preferences).
    • Cơ chế Ghi nhận (Write): Chỉ trích xuất tự động các sự kiện cốt lõi (Facts) và phong cách giao tiếp yêu thích (Preferences) để lưu vào database (không lưu lịch sử chat thô để tránh phình Context).
    • Cơ chế Nạp (Read): Frontend tự động truyền user_id khi đăng nhập. Node khởi đầu (MEMORY) tự động truy vấn CSDL Postgres và nạp hồ sơ người dùng tương ứng vào state["user_profile"].
    • Giải quyết mâu thuẫn bộ nhớ (Memory Conflict Resolution): Cuối phiên chat, một Memory Manager Agent (LLM) sẽ chạy ngầm so sánh thông tin cũ trong CSDL và thông tin mới thu thập được. Nếu có mâu thuẫn trực tiếp, nó sẽ tự động cập nhật/ghi đè theo thông tin mới nhất và lưu kèm timestamp.
  • Structured Output (JSON Schema) bắt buộc:
    • Định nghĩa cục bộ: Cấu trúc JSON Schema (Pydantic BaseModel) bắt buộc phải được định nghĩa trực tiếp ở đầu tệp tin của chính Agent sử dụng nó (Ví dụ: agents/query_reform.py), tuân thủ nguyên tắc Cô lập chức năng (Modularity) để dễ bảo trì.
    • Adapter trung gian: Lớp model adapter trong config.py làm nhiệm vụ nhận schema và cấu hình trực tiếp vào API của nhà cung cấp (Google GenAI SDK hoặc OpenAI JSON Mode) để đảm bảo mô hình luôn trả về JSON hợp lệ 100%.
  • Cơ chế gộp dữ liệu song song (Parallel Reducers):
    • LangGraph sử dụng các hàm reducer (như merge_dict, merge_list) trong state.py để gộp lũy kế kết quả tuần tự theo từng cặp từ các Agent chạy song song (Fan-out/Fan-in) trở lại vào Global State. Cơ chế này hỗ trợ mở rộng không giới hạn số lượng Agent chạy song song.
  • Định dạng file code mới: Mọi tệp tin mới được khởi tạo phải bắt đầu bằng Copyright Header ghi rõ: 
    # Chức năng: [Chức năng file]
